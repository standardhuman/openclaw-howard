## 🔴 Action Needed

### Qwen Portal Auth Removed (Breaking)
The deprecated `qwen-portal-auth` OAuth integration for portal.qwen.ai is gone. Must migrate to Model Studio with `openclaw onboard --auth-choice modelstudio-api-key`.
- **Impact on us:** None — we don't use Qwen.

### Old Config Migrations Dropped (Breaking)
Auto-migrations for config keys older than two months now fail validation instead of being silently rewritten.
- **Impact on us:** Low — we're on a recent version. Run `openclaw doctor` to verify.

## 🟡 Worth Exploring

### Plugin Approval Hooks (#55339)
New `requireApproval` in `before_tool_call` hooks — plugins can now pause tool execution and prompt user approval via Telegram buttons, Discord interactions, or `/approve`. Our `/approve` command auto-handles both exec and plugin approvals now.
- **Impact:** Could be useful for gating sensitive plugin actions.

### xAI/Grok Web Search via Responses API (#56048)
Bundled xAI provider now uses Responses API with first-class `x_search`. Auto-enables from web-search config.
- **Impact:** If we ever try Grok, search is now built-in.

### ACP Chat Binds for Discord/BlueBubbles/iMessage
`/acp spawn codex --bind here` can now turn the current chat into a Codex workspace without a child thread.
- **Impact:** Useful for in-chat ACP workflows on Discord.

### Matrix TTS Voice Bubbles (#37080)
Auto-TTS replies now send as native Matrix voice bubbles instead of generic audio attachments.
- **Impact:** Direct improvement for our Matrix channel — TTS will sound/look better.

### Slack `upload-file` Action + Teams/Google Chat File Sends
Explicit file upload action for Slack, Teams, and Google Chat.
- **Impact:** None currently — we don't use these channels.

### Memory Plugin Flush Refactor
Pre-compaction memory flush now owned by `memory-core` plugin contract instead of hardcoded logic.
- **Impact:** Should be transparent, but worth monitoring for memory behavior changes.

## 🟢 FYI

### Bug Fixes Relevant to Us
- **Anthropic stop reason crash fix (#56639):** Unhandled provider stop reasons (e.g. `sensitive`) now produce structured errors instead of crashing. Stability improvement.
- **Gemini 3.1 model resolution (#56567):** Fixed flash-lite prefix ordering for Google provider aliases.
- **Telegram message splitting (#56595):** Long messages now split at word boundaries, not mid-word. Nice QoL fix.
- **Telegram empty text crash (#56620):** Whitespace-only replies no longer crash with GrammyError 400.
- **Telegram replyTo validation (#56587):** Invalid replyToMessageId strings no longer cause errors.
- **NO_REPLY envelope suppression (#56612):** JSON-wrapped `{"action":"NO_REPLY"}` no longer leaks to channels.
- **Agents/compaction fixes (#49479, #51072):** Post-compaction AGENTS refresh preserved; benign `/compact` no-ops labeled "skipped" not "failed."

### Other Changes
- **MiniMax image generation:** New image-01 model provider (generate + image-to-image editing).
- **OpenAI `apply_patch`:** Enabled by default for OpenAI/Codex models.
- **Bundled CLI plugins auto-load:** Claude CLI, Codex CLI, Gemini CLI no longer need manual `plugins.allow` entries.
- **`openclaw config schema`** command added — prints JSON schema for `openclaw.json`.
- **Zsh completion fix (#56555):** `compdef` deferred until `compinit` is available.
- **Control UI config safety:** Sensitive config hidden by default, reveal-to-edit state restored.
