## 🟡 Worth Exploring

### New `PermissionDenied` Hook (v2.1.88)
Fires after auto mode classifier denials. Return `{retry: true}` to let the model retry.
**Relevance:** If we ever adopt auto mode for interactive Claude Code sessions (instead of bypass permissions), this hook would let us build custom retry logic. Not urgent but architecturally interesting.

### Thinking Summaries Disabled by Default (v2.1.88)
Thinking summaries no longer generated in interactive sessions. Set `showThinkingSummaries: true` in settings to restore.
**Relevance:** If Brian uses Claude Code interactively and misses thinking summaries, re-enable via settings. Our OpenClaw agents (headless) aren't affected.

### Named Subagents in `@` Mention Typeahead (v2.1.88)
You can now `@`-mention named subagents in the typeahead suggestions.
**Relevance:** Nice UX improvement for interactive Claude Code usage with multi-agent setups.

### Hooks `if` Condition Fix (v2.1.88)
Fixed `if` condition filtering not matching compound commands (`ls && git push`) or commands with env-var prefixes (`FOO=bar git push`).
**Relevance:** If we use hooks with conditional filtering, this could affect behavior. Check any hook configs.

### PreToolUse/PostToolUse Hooks: Absolute Paths (v2.1.88)
`file_path` is now provided as an absolute path for Write/Edit/Read tools.
**Relevance:** Any existing hook scripts parsing relative paths may need adjustment (unlikely for our setup, but worth noting).

### `/stats` Now Includes Subagent/Fork Usage (v2.1.88)
Previously undercounted tokens by excluding subagent/fork usage. Now counts everything.
**Relevance:** Token usage numbers in `/stats` will jump — this is more accurate, not a cost increase.
