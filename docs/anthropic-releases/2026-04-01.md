## 🔴 Action Needed

### Claude Code v2.1.89 Released (April 1) — We're On It ✅
Local version confirmed: **2.1.89** (native installer, NOT npm).

**New features in v2.1.89 (on top of v2.1.88 from yesterday):**

**`"defer"` permission decision for PreToolUse hooks (🔴 for OpenClaw):**
- Headless sessions can now **pause at a tool call** and resume with `-p --resume` to have the hook re-evaluate.
- This is directly relevant to OpenClaw's headless agent architecture. We can now build workflows where agents pause for human approval on specific operations, then resume.
- **Action:** Evaluate whether any of our 43 agents would benefit from deferred permission handling instead of bypass-all.

**Autocompact thrash loop fix (critical stability):**
- Fixed a bug where context refilling immediately after compacting (3 times in a row) would burn API calls in an infinite loop. Now stops with an actionable error.
- This was likely causing silent token waste in our long-running agent sessions.

**MCP connection non-blocking for `-p` mode:**
- `MCP_CONNECTION_NONBLOCKING=true` skips the MCP connection wait entirely for print mode.
- `--mcp-config` server connections bounded at 5s instead of blocking on the slowest server.
- **Action:** Consider setting `MCP_CONNECTION_NONBLOCKING=true` for our headless agent spawns to improve startup time.

**Edit tool improvement:**
- `Edit` now works on files viewed via `Bash` with `sed -n` or `cat`, without requiring a separate `Read` call first.
- Reduces token usage by eliminating redundant Read calls.

**Hook output size limit:**
- Hook output over 50K characters is now saved to disk with a file path + preview instead of being injected directly into context.
- Prevents context pollution from verbose hook output.

**Bash tool file-modification warning:**
- Warns when a formatter/linter command modifies files you've previously read, preventing stale-edit errors.

**`/buddy` — April Fools' Easter egg:**
- Tamagotchi-style terminal pet with stats like CHAOS and SNARK. Fun, not actionable.

## 🟡 Worth Exploring

### Batch API: 300k Output Tokens (New!)
The models overview page now confirms: **Opus 4.6 and Sonnet 4.6 support up to 300k output tokens** on the Message Batches API using the `output-300k-2026-03-24` beta header.

**Relevance:** If any OpenClaw workflows use batch processing for long-form generation, this is a 2.3x increase over the previous 128k ceiling. Worth evaluating for heavy document generation or code analysis tasks.

### Symlink Security Fix (v2.1.89)
`Edit(//path/**)` and `Read(//path/**)` allow rules now check the **resolved symlink target**, not just the requested path. This prevents symlink-based permission bypasses.

**Relevance:** If any of our agent workspaces use symlinks, verify they still work with updated path resolution.

### `cleanupPeriodDays: 0` Now Rejected (v2.1.89)
Previously silently disabled transcript persistence. Now throws a validation error.

**Relevance:** Check if any of our configs set this to 0.
