## Matrix Plugin Debugging (10:00 AM - 10:44 AM)

**Problem:** Brian messaging Howard on Element/Matrix — no response. Matrix plugin not loading after gateway restarts.

**Root Cause Chain (multiple issues discovered):**
1. **Matrix plugin v2026.3.7 installed but incompatible** — imports `openclaw/plugin-sdk/matrix` which doesn't exist in any OpenClaw version. The working v2026.3.2 imports `openclaw/plugin-sdk` (no `/matrix` suffix).
2. **Two OpenClaw installs on the system:**
   - `/opt/homebrew/lib/node_modules/openclaw` — v2026.3.8 (updated via `npm install -g`, has `keyed-async-queue` export)
   - `/Users/brian/.local/share/fnm/node-versions/v22.22.0/installation/lib/node_modules/openclaw` — **still v2026.2.15** (old, missing exports)
3. **LaunchAgent plist mismatch:** Service uses `/opt/homebrew/bin/node` but runs the **fnm** OpenClaw entry point (`v22.22.0/.../openclaw/dist/index.js`). So gateway runs v2026.2.15 code, which lacks `keyed-async-queue` and `plugin-sdk/matrix` exports.
4. **Duplicate plugin conflict:** Both `/opt/homebrew/.../extensions/matrix/` and `~/.openclaw/extensions/matrix/` existed → "duplicate plugin id" warning, second one overridden.

**Fixes Applied:**
- Updated OpenClaw to v2026.3.8 via `npm install -g openclaw@2026.3.8` (went to /opt/homebrew)
- Downgraded matrix plugin to v2026.3.2 (clean `npm pack` + extract) at `~/.openclaw/extensions/matrix/`
- Removed duplicate matrix extension from `/opt/homebrew/.../extensions/matrix/`
- **Fixed LaunchAgent plist** — changed gateway entry point from fnm path to `/opt/homebrew/lib/node_modules/openclaw/dist/index.js`
- Updated plist version comment and SERVICE_VERSION to 2026.3.8
- Ran `launchctl bootout` + `bootstrap` to restart service with new plist
