## 💡 AI Business Opportunities

### Karpathy's Autoresearch — The "Overnight Experiment Lab" Pattern
- **Source:** https://github.com/karpathy/autoresearch (105 pts on HN)
- **Idea:** Karpathy's new repo lets an AI agent autonomously run ML experiments: modify code, train for 5 minutes, evaluate, keep or discard, repeat. ~100 experiments while you sleep. The architecture is stunningly simple — the agent only modifies `train.py`, guided by `program.md` (which Karpathy explicitly calls a "skill"). The human's job is to iterate on the program.md — "programming the program," not writing code. Karpathy's tongue-in-cheek README: "Research is now entirely the domain of autonomous swarms of AI agents... the 'code' is now a self-modifying binary that has grown beyond human comprehension. This repo is the story of how it all began."
- **Why now:** This pattern generalizes beyond ML research. Any domain where you can define a metric and a 5-minute experiment cycle is a candidate: marketing copy testing (metric: click-through rate), pricing optimization (metric: conversion rate), even service report template iteration (metric: customer response rate). Brian could adapt this pattern for SailorSkills — give an agent the Pro app UI, a satisfaction metric, and let it iterate on layout/copy overnight.
- **Effort to test:** Medium — requires defining a clear metric and experiment harness. Start with something simple like email subject line testing or landing page variant generation.

### Verification Debt — The Business Opportunity in AI Code Quality
- **Source:** https://fazy.medium.com/agentic-coding-ais-adolescence-b0d13452f981 (94 pts on HN)
- **Idea:** Lars Janssen coins "verification debt" — the growing gap between how fast we can generate output and how fast we can validate it. His key insight: "if AI makes every engineer even 50% more productive, the org doesn't get 50% more output. It gets 50% more pull requests, 50% more documentation... and someone has to review all of it." The bottleneck moves upstream to judgment, domain understanding, and risk assessment. The HN discussion (83 comments) reveals widespread agreement that this is the defining challenge of 2026 — not "how do we produce more code?" but "how do we verify more code?"
- **Why now:** This connects to our March 4 report on the verification gap (Leo de Moura) and March 7's Simon Willison agentic testing guide. The pattern is solidifying: tools that help verify AI output are becoming more valuable than tools that help generate it. A "verification-as-a-service" offering — reviewing AI-generated PRs, auditing agent outputs, certifying AI-built code — is a real business opportunity. Kevin Browne's companion piece (kevinbrowne.ca) formalizes this as "the AI era's technical debt."
- **Effort to test:** Low — Brian is already living this with his agent team. Document the verification workflow (how Howard, Reese, and the sub-agents review each other's work) as a case study. That documentation itself is valuable content.

### "Files Are the Interface" — Validates OpenClaw's Architecture
- **Source:** https://madalitso.me/notes/why-everyone-is-talking-about-filesystems (207 pts on HN, 117 comments)
- **Summary:** Daniel Phiri (ex-Weaviate) documents the emerging consensus: LlamaIndex, LangChain, Oracle, Dan Abramov, Archil, and Karpathy all converging on filesystems as the primary interface for AI agents. Key arguments: (1) Context windows aren't memory — files are. (2) The file format IS the API — SKILL.md, AGENTS.md, aboutme.md are all interoperability without coordination. (3) Agent Skills (agentskills.io) has become an open standard adopted by Microsoft, OpenAI, Atlassian, GitHub, and Cursor. (4) NanoClaw's "skills over features" model proves that any capability can be a markdown file. Critical caveat from ETH Zürich research: overly long context files actually reduce agent task success and increase costs by 20%+. The fix isn't "no context files" — it's "minimal context files."
- **Action:** Three things for Brian: (1) Audit AGENTS.md and CLAUDE.md length — trim to essential constraints only. The ETH Zürich finding suggests Brian's current files may be too verbose for optimal agent performance. (2) This essay is prime content for a blog post: "How our marine service business runs on markdown files" — showing SailorSkills' OpenClaw setup as a real-world example of the filesystem-first thesis. (3) The aboutme.md concept (portable identity descriptor) could become a SailorSkills Pro feature — each diver gets a profile markdown file that any tool can read.

### Zip Code First — UX Pattern for SailorSkills Forms
- **Source:** https://zipcodefirst.com (336 pts, 250 comments on HN)
- **Summary:** A viral rant-turned-demo about address form UX: put the zip code first, and you can autofill city, state, and country from 5 keystrokes. The HN discussion (250 comments!) reveals massive frustration with form UX across the industry. The broader principle: ask for the most information-dense field first, derive everything else.

### LLM Writing Tropes — AI Slop Detection Directory
- **Source:** https://tropes.fyi (170 pts, 69 comments on HN)
- **What it does:** A directory of recurring patterns that give away AI-generated text, with tools to detect and fix them. Includes: "ai;dr" (summarize the prompt that generated a piece of slop), "AI Vetter" (drop a link, get told if it's worth reading), "Deslopify" (paste text, get a diff of AI patterns with rewrites), and a downloadable system prompt file to prevent these patterns. The creator's manifesto: "Do not waste your time reading words that nobody wrote."
- **How we'd use it:** Two applications: (1) Run SailorSkills' marketing copy, service reports, and client communications through Deslopify to catch AI tells. Brian's brand depends on authenticity — AI slop in client-facing content would undermine the trust advantage. (2) Add the anti-tropes system prompt to OpenClaw agents that produce client-facing text (service reports, marketing materials). Prevent the problem at the source.
